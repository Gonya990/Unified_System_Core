# 💎 Архитектура системы "Vibranium Logic"

Этот документ описывает "кремниевую" логику — систему, которая спроектирована быть отказоустойчивой, прозрачной и самовосстанавливающейся.

## 1. Иерархия Агентов (Consilium)

Все агенты в системе имеют фиксированные идентификаторы и роли. Изменение имен запрещено.

| Агент | Роль | Программа | Проект (Slug) |
| :--- | :--- | :--- | :--- |
| **PinkLake** | Верховный Оркестратор | `antigravity-core` | `home-kosta-documents-unified-system-core` |
| **OrangeStone** | Главный Технический Агент (Antigravity) | `antigravity` | `home-kosta-documents-unified-system-core` |
| **FuchsiaCat** | Операционный агент Кости | `opencode` | `home-kosta-documents-unified-system-core` |

## 2. Железная Синхронизация (full_sync.sh)

Скрипт синхронизации — это единственный "рубильник". Он выполняет:

1. **Локальную очистку**: Проверка состояния Git, сохранение изменений.
2. **Агрессивное обновление сервера**: `git fetch` + `git reset --hard` (гарантирует отсутствие конфликтов на сервере).
3. **Восстановление подмодулей**: Автоматическое исправление ссылок Git.
4. **Перезапуск в Docker**: Сборка и запуск сервисов через `docker compose`.
5. **Пост-верификацию**: Проверка логов на наличие ошибок `ERROR` или `ValueError`.

## 3. Распределенный Интеллект (Distributed AI)

Логика маршрутизации в `inference_client.py` разделяет задачи:

* **Linux (Titan RTX)**: Тяжелые модели, анализ изображений, длинные контексты.
* **Windows (Igor Node)**: Легкие и быстрые модели (Llama 3.2), TTS.

Если один из узлов недоступен, система автоматически переключается на доступный (Failover).

## 4. Защита от Сбоев (Guard Rails)

1. **Проверка команд**: Запрет на нелатинские команды в Telegram (предотвращает крах Bot API).
2. **Изоляция зависимостей**: Строгое использование `venv` и разделение имен (например, `notion_service.py` вместо `notion_client.py`).
3. **Watchdog**: Docker всегда настроен на `restart: always`.

## 5. Коммуникации (MCP Mail)

Все общение между агентами идет через MCP Mail.

* **Threads**: Каждая задача имеет свой `thread_id` (например, `SYNC-2026-01-09`).
* **ACK**: Каждое важное сообщение требует подтверждения (`ack_required: true`).
