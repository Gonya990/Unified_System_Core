# Distributed Agent Billboard Specification

## Concept
The Billboard is a decentralized task coordination mechanism. Instead of a centralized orchestrator polling agents or agents polling a central database, agents publish and subscribe to tasks on the Billboard.

## Architecture
- **Billboard (Message Bus / Pub-Sub)**: Built on Redis or PostgreSQL Listen/Notify.
- **Tasks**: JSON objects containing `task_id`, `type`, `payload`, `status`, `assigned_to`.
- **Agents**: Autonomous workers that listen to the Billboard for tasks matching their capabilities.

## Flow
1. User or another agent posts a task to the Billboard.
2. Agents subscribed to the relevant task type receive a notification.
3. The first available and capable agent claims the task (updates `assigned_to`).
4. The agent executes the task and updates the status to `COMPLETED` or `FAILED`.
5. The result is stored in the Knowledge Graph or `rep/claims/` directory.

## Implementation details
- `/billboard/tasks/`: Directory for file-based task queuing (if using file-based approach).
- `/contexts/`: Stores the context required for agents to execute tasks.
- `/rep/claims/`: Stores the output or claims generated by agents.
