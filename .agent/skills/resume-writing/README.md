# Resume Writing Skills

> **Location:** `.agent/skills/resume-writing/`
> **Purpose:** Comprehensive guide for crafting professional, ATS-optimized resumes

## Quick Reference

### The 6-Second Rule
Hiring managers spend an average of **6 seconds** looking at a resume before deciding if it's worth reading. Make every word count.

### Key Statistics
- Tailored resumes are **6X more likely** to land interviews than generic ones
- 75% of resumes are rejected by ATS before reaching human eyes
- Quantified achievements increase callback rates by 40%

---

## Resume Structure

### Recommended Format: Reverse Chronological
The reverse-chronological format remains the gold standard because:
- Clear and familiar to recruiters
- Easy for ATS to scan
- Shows career progression naturally

### Essential Sections (In Order)

1. **Contact Information** - Name, phone, email, LinkedIn, location (city/state)
2. **Professional Summary** - 60-100 word elevator pitch (optional if using cover letter)
3. **Work Experience** - Most recent first, 3-6 bullet points per role
4. **Skills** - Technical skills, tools, certifications
5. **Education** - Degrees, relevant coursework, GPA if >3.5

## Tailoring Your Resume

### Step 1: Analyze the Job Description
- Highlight required skills, qualifications, and experience
- Identify specific keywords (e.g., "project management," "data analysis")
- Note the company's language and tone

### Step 2: Mirror Keywords
Include exact phrases from the job posting:
- If they say "cross-functional collaboration" → use that phrase
- If they say "stakeholder management" → don't substitute "working with people"

### Step 3: Reorder Bullet Points
- Place most relevant experience at the top of each role
- Move matching skills to prominent positions
- Customize your summary for each application

### Step 4: Optimize for ATS
```
DO:
✓ Use standard section headers (Experience, Education, Skills)
✓ Include keywords naturally in context
✓ Use simple formatting (no tables, graphics, columns)
✓ Submit as .docx when possible (better ATS compatibility)

DON'T:
✗ Stuff keywords unnaturally
✗ Use headers/footers for important info
✗ Use fancy templates with graphics
✗ Submit PDFs created in Canva/InDesign (ATS struggles to parse)
```

## Writing Powerful Bullet Points

### Formula 1: PAR (Problem-Action-Result)
Best for: Accomplishments, technical skills, problem-solving

```
Structure: [Problem/Context] → [Action taken] → [Quantified Result]

Example:
"Reduced customer support response time by 20% by implementing
a new ticketing system and optimizing workflow processes."
```

### Formula 2: STAR (Situation-Task-Action-Result)
Best for: Behavioral demonstrations, detailed accomplishments

```
Structure: [Situation] → [Task] → [Action] → [Result]

Example:
"Managed inventory levels in a fast-paced retail setting;
achieved a 15% reduction in inventory costs through
strategic reordering and vendor negotiations."
```

### Formula 3: Result-First
Best for: Impressive metrics, grabbing immediate attention

```
Structure: [Quantified Result] → [Action/Method]

Example:
"Increased sales revenue by 18% through targeted marketing
campaigns and strategic account management."
```

### Formula 4: Action + Skill + Result
Best for: Technical/engineering roles, concise descriptions

```
Structure: [Action Verb] + [Skill/Tool] + [Result]

Example:
"Developed user-friendly website using React and Node.js,
resulting in a 30% increase in user engagement."
```

## Power Action Verbs

### Leadership
Directed, Orchestrated, Spearheaded, Championed, Pioneered, Mobilized

### Achievement
Exceeded, Surpassed, Outperformed, Accelerated, Maximized, Transformed

### Creation
Engineered, Architected, Designed, Developed, Established, Launched

### Improvement
Streamlined, Optimized, Revitalized, Modernized, Consolidated, Enhanced

### Analysis
Assessed, Evaluated, Analyzed, Investigated, Diagnosed, Forecasted

### Communication
Negotiated, Persuaded, Presented, Collaborated, Facilitated, Mediated

## Quantifying Achievements

### Always Try to Include:
- **Percentages**: "Increased efficiency by 35%"
- **Dollar amounts**: "Managed $2.5M budget"
- **Time savings**: "Reduced processing time from 3 days to 4 hours"
- **Volume/Scale**: "Processed 500+ transactions daily"
- **Team size**: "Led cross-functional team of 12"
- **Rankings**: "Ranked #1 in sales region for Q3"

### When You Don't Have Exact Numbers:
- Use ranges: "Managed 50-75 client accounts"
- Use approximations: "Approximately 200 stakeholders"
- Use scope: "Company-wide initiative across 5 departments"

## Common Mistakes to Avoid

### From Hiring Managers Who Reviewed 1000+ Resumes:

#### 1. Listing Duties Instead of Achievements
```
BAD:  "Responsible for managing social media accounts"
GOOD: "Grew Instagram following by 150% in 6 months, generating
       $50K in attributed revenue through strategic content campaigns"
```

#### 2. Being Generic
```
BAD:  "Team player with excellent communication skills"
GOOD: "Presented quarterly performance updates to C-suite
       executives and cross-functional teams of 25+ stakeholders"
```

#### 3. Failing to Show Impact
```
BAD:  "Worked on various projects"
GOOD: "Delivered 8 client projects on time and under budget,
       resulting in 95% client retention rate"
```

### Other Critical Mistakes:
- Including Word/Excel as "skills" (assumed knowledge)
- Exaggerating responsibilities (reviewers know typical intern scope)
- Using overly flowery language (simple is always better)
- Including irrelevant personal information
- Typos or grammatical errors (instant rejection)
- Inconsistent formatting or tense usage

## Resume Length Guidelines

| Experience Level | Recommended Length |
|-----------------|-------------------|
| Entry-level / New Grad | 1 page |
| Mid-level (3-10 years) | 1-2 pages |
| Senior / Executive | 2-3 pages |
| IT / Cybersecurity / Government | 2-3 pages |
| Academic CV | 5-12 pages |

**Golden Rules:**
- Never 1.5 pages (either 1 or 2)
- Most recent role gets the most bullet points
- Decrease detail as you go back in time

## Professional Summary

### When to Include:
- Not submitting a cover letter
- Changing careers (need to explain pivot)
- Have a unique value proposition

### When to Skip:
- Already including a tailored cover letter
- Space is limited and experience speaks for itself

### Structure (60-100 words):
```
[Professional Identity] with [X years] experience in [Industry/Field].
Proven track record of [Key Achievement #1] and [Key Achievement #2].
Skilled in [Relevant Skill 1], [Relevant Skill 2], and [Relevant Skill 3].
Seeking to leverage expertise in [Target Role/Company Goal].
```

### Example:
```
Results-driven Marketing Manager with 7+ years of experience in B2B SaaS.
Increased qualified leads by 200% and reduced customer acquisition cost
by 35% through data-driven campaign optimization. Expertise in marketing
automation, content strategy, and cross-functional team leadership.
Seeking to drive growth at a high-impact technology company.
```

## Skills Section

### Technical Skills - DO Include:
- Programming languages (with proficiency implied)
- Software/tools specific to your field
- Industry certifications
- Specialized methodologies (Agile, Six Sigma, etc.)

### Technical Skills - DON'T Include:
- Microsoft Word/Excel (assumed unless specifically requested)
- "Proficient in email"
- Self-rated skill levels (unless industry standard)

### Soft Skills - Handle Differently:
**Never list these directly:**
- "Team player," "Hard worker," "Detail-oriented"

**Instead, demonstrate through bullet points:**
```
Instead of: "Excellent communication skills"
Write: "Presented technical findings to non-technical stakeholders,
        securing $500K in additional project funding"
```

## Education Section

### Placement:
- **Recent graduates**: Top of resume, above experience
- **Experienced professionals**: Below experience section
- **Career changers**: Depends on relevance of degree

### Include:
- Degree, major, institution, graduation date
- GPA if >3.5 (or >3.0 for new grads if competitive)
- Relevant coursework (3-4 courses max)
- Academic honors, Dean's List
- Relevant projects with measurable outcomes

### Skip After 1-2 Years of Work:
- GPA
- Coursework details
- High school information

## Handling Employment Gaps

### Be Transparent but Brief:
```
Personal Leave of Absence | Jan 2022 - Aug 2022
- [Optional: Brief, professional explanation]
```

### Acceptable Gap Explanations:
- Family caregiving
- Health-related leave
- Professional development / Certifications
- Freelance / Consulting work
- Volunteer work

### Frame Productively:
```
Career Sabbatical | 2022
Completed AWS Solutions Architect certification while
providing care for family member. Maintained industry
knowledge through professional development courses.
```

## For Limited Experience (Students/New Grads)

### Lead with Education:
- Place above experience section
- Include relevant coursework
- Highlight academic projects with outcomes

### Maximize Projects:
```
E-Commerce Website Development | React, Node.js, MongoDB
- Built full-stack application with user authentication,
  shopping cart, and payment integration
- Achieved 200+ orders in first month of deployment
- GitHub: [link] | Live Demo: [link]
```

### Reframe Part-Time Work:
```
BAD:  "Made coffee and served customers"
GOOD: "Maintained 98% customer satisfaction rating while
       processing 200+ daily transactions during peak hours"
```

### Include Relevant Activities:
- Leadership in student organizations
- Volunteer work with measurable impact
- Academic competitions or hackathons
- Research assistance

## Application Strategy

### Quality vs. Quantity Balance:

| Career Stage | Time per Application |
|-------------|---------------------|
| Student | 30+ minutes |
| New Graduate | 1-2 hours |
| Experienced | 2-4 hours |

### Resume Variants to Maintain:
1. **Industry-specific** - Tailored keywords and emphasized skills
2. **Company size** - Technical depth varies (startup vs. enterprise)
3. **Role-specific** - Highlight different aspects of experience

### Application Tracking:
- Keep spreadsheet of applications sent
- Note which resume version used
- Track response rates to optimize

## File Format & Submission

### Recommended: Word Document (.docx)
- Better ATS compatibility
- Easier for recruiters to annotate
- Use when format not specified

### When to Use PDF:
- Explicitly requested
- Sending directly to hiring manager (not through ATS)
- Design/creative roles where formatting matters

### Avoid:
- PDFs from Canva, InDesign, or LaTeX (ATS parsing issues)
- Google Docs links (send downloaded file)
- Creative formats with columns/graphics

## Cover Letters

### When Required:
- Job posting specifically requests one
- Applying to highly competitive position
- You have context that doesn't fit resume format
- Personal connection to company

### Structure (Concise):
1. **Introduction** - Position applying for, how you found it
2. **Value proposition** - 2-3 relevant qualifications
3. **Specific interest** - Why this company specifically
4. **Call to action** - Request for interview

### Key Points:
- Keep to one page (3-4 paragraphs)
- Address to specific person when possible
- Different document from resume (don't combine)
- Email body is NOT your cover letter (attach separately)

## Final Checklist

### Before Submitting:

```
□ Tailored to specific job description
□ Keywords from posting included naturally
□ All bullets show impact/achievements (not just duties)
□ Quantified results where possible
□ Consistent formatting throughout
□ Correct tense (present for current role, past for previous)
□ No typos or grammatical errors
□ Contact information complete and professional
□ File named professionally (FirstName_LastName_Resume.docx)
□ Saved in correct format (.docx or .pdf as required)
```

## Templates

### Basic Bullet Point Template:
```
[Action Verb] + [What You Did] + [Tool/Method] + [Quantified Result]

"Streamlined invoice processing workflow using automated scripts,
reducing processing time by 60% and saving 15 hours weekly."
```

### Summary Template:
```
[Title] with [X] years of experience in [Industry].
Proven ability to [Achievement 1] and [Achievement 2].
Expertise in [Skill 1], [Skill 2], and [Skill 3].
```

### Project Description Template:
```
[Project Name] | [Technologies Used]
- [What you built] + [its purpose]
- [Your specific role/contribution]
- [Quantified outcome or impact]
- [Link to demo/code if applicable]
```

## Quick Wins Checklist

For immediate resume improvement:

1. **Replace "Responsible for" with action verbs**
2. **Add at least one number to every bullet point**
3. **Remove Word/Excel from skills section**
4. **Cut any bullet that doesn't show impact**
5. **Ensure most relevant experience appears first**
6. **Match 5+ keywords from target job description**
7. **Remove objective statement (use summary or nothing)**
8. **Delete "References available upon request"**
