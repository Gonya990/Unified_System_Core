## 🔍 Проблемные Интеграции

### 1. ❌ SmartThings (КРИТИЧНО)

- **Статус:** Failed
- **Ошибка:** `Reached limit of subscriptions`
- **Затронуто:** 27 устройств (предположительно)
- **Решение:**
  1. Перейти на <https://smartthings.developer.samsung.com/>
  2. Войти с учетной записью Samsung
  3. Удалить старые неиспользуемые подписки (webhooks)
  4. Или создать новый Personal Access Token
  5. В HA: удалить интеграцию SmartThings
  6. Добавить заново с новым токеном

### 2. ❌ Bluetooth

- **Статус:** Failed to set up
- **Ошибка:** `DBus service not found`
- **Причина:** Docker контейнер не имеет доступа к DBus хоста
- **Решение:**

  ```yaml
  # В docker-compose.yaml добавить:
  volumes:
    - /run/dbus:/run/dbus:ro
  network_mode: host
  ```

  Затем: `docker-compose down && docker-compose up -d`

### 3. ❌ Ollama

- **Статус:** Failed to set up
- **Причина:** Сервер Ollama недоступен или неправильный адрес
- **Решение:**
  1. Проверить, запущен ли Ollama на целевой машине
  2. Убедиться, что IP/порт в конфигурации HA правильные
  3. Проверить доступность через: `curl http://IP:PORT/api/tags`

### 4. ⚠️ LocalTuya

- **Статус:** Частичные ошибки
- **Причина:** Изменились `local_key` устройств
- **Решение:**
  1. Получить новые `local_key` через Tuya IoT Platform
  2. Обновить конфигурацию устройств в LocalTuya
  3. Перезагрузить интеграцию
