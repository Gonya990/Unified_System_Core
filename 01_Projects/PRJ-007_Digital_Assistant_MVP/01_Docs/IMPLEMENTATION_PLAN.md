# PRJ-007: Digital Assistant MVP - Implementation Plan

## 🎯 Objective

Create a scalable "Business Control Center" that automates lead generation,
client communication (WhatsApp), and payments (Crypto) for small businesses
in Israel.

---

## 🏗 Phase 1: Neural Hub (Make.com / Activepieces)

We will use **Make.com** as the orchestration layer to connect
disparate systems.

### 1.1 Connectivity Schema

1. **Entry Point**: WhatsApp Business (Meta API) or Telegram.
2. **Logic Node**: Webhook triggers our **AI Core (Ollama/Gemini)**.
3. **Memory**: SQLite or Google Sheets for CRM.
4. **Action**: Automated follow-up and appointment scheduling (Google Calendar).

### 1.2 "Control Center" Setup Steps

- [ ] Create Make.com account.
- [ ] Set up **Custom Webhook** in Make.
- [ ] Connect our AI Bot via REST API (exposed through Tailscale SSH tunnel).
- [ ] Integrate Google Sheets for "Lead Tracking".

---

## 🚀 Phase 2: Power Demo (Marketing Offer)

**The Offer**: *"Automated Digital Manager that lives in your WhatsApp. It
identifies leads, answers questions, and closes deals while you sleep."*

### 2.1 Target Market

- Real Estate agents.
- Home service businesses.
- Freelance developers/consultants.

### 2.2 Visual Assets

- Created `vision_dashboard.html`: A premium dashboard representing the state
  of the system.
- Need: High-quality screen recordings of the bot in action.

---

## 💎 Phase 3: Crypto Gateway (Capital Sovereignty)

### 3.1 Requirements

- **USDT (TRC20)** support: Preferred for low fees and stability.
- **BTC/ETH** support: For high-value transactions.
- **API Integration**: Connect to `NowPayments` or a local node via GPU server.

### 3.2 Workflow

- Invoice generated by AI Bot.
- QR code sent to WhatsApp.
- Payment confirmation via Webhook -> Product delivery/Access granted.
