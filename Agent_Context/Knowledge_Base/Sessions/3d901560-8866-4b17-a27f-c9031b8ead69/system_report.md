### 🛠 Как я это делаю

* **Инфраструктура как код (IaC)**: Все настройки хранятся в скриптах (`deploy.exp`, `setup.ps1`), что позволяет восстановить систему за минуты.

## 3. Security Analysis (Анализ безопасности)

### 🛡 Уязвимости (Vulnerability Assessment)

#### Сильные стороны (Strong Points)

* **Нет открытых портов**: Внешний мир не видит ваши порты (22, 80, 8006). Все закрыто за Tailscale VPN.
* **SSH Key Auth**: На новой VM (`106`) вход только по ключу. Брутфорс паролей бесполезен.
* **Изоляция**: Эксперименты идут в VM или VirtualEnv. Основная система (Proxmox Host) чиста.

#### Слабые места (Weak Points)

1. **Windows SSH (Password Auth)**:
    * *Проблема*: На Windows вход всё еще по паролю (`GarYk...`).
    * *Риск*: Если злоумышленник попадет в Tailscale сеть, он сможет подбирать пароль.
    * *Решение*: Настроить `administrators_authorized_keys` на Windows OpenSSH (требует прав админа).
2. **Хранение ключей (Local Storage)**:
    * *Проблема*: Приватный ключ `id_rsa_proxmox` лежит просто в папке `Sandbox`.
    * *Риск*: Если украдут MacBook, получат доступ ко всему.
    * *Решение*: Использовать менеджер паролей (1Password SSH Agent) или зашифровать ключ парольной фразой.
3. **Tailscale ACLs**:
    * *Проблема*: Сейчас "все видят всех" (Implicit Allow).
    * *Решение*: В будущем настроить ACL в Tailscale, чтобы, например, Windows не могла ходить на Proxmox, а только наоборот.

### Итог по безопасности: **ВЫСОКАЯ (для домашней сети)**

Благодаря Tailscale вы защищены от 99% атак из интернета (сканеры портов, ботнеты). Основной вектор угрозы — компрометация самого MacBook или учетной записи Google/GitHub.
