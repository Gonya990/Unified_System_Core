# Обзор интеграции Gemini

## Краткое содержание
В проект интегрирована модель Gemini 2.0 Flash с использованием Python-структуры. Интеграция включает схему конфигурации, реализацию клиента и реестр инструментов.
Дополнительно реализован веб-интерфейс на Flask для визуального взаимодействия с моделью.
Добавлены возможности SSH для управления хостами NUC и WSL.

## Изменения
### Конфигурация
- Создан `config/control_schema.json` с API ключом, настройками модели и конфигурацией хостов.
- Добавлен `flask` в `requirements.txt`.

### Исходный код
- `src/gemini.py`: Реализует класс `GeminiClient` для обработки API запросов.
- `src/ssh_instruments.py`: Реализует инструменты управления SSH ключами и выполнением команд.
- `src/instruments.py`: Реализует `InstrumentRegistry` для управления и публикации инструментов (Gemini + SSH).
- `main.py`: Скрипт точки входа для демонстрации использования через CLI.
- `app.py`: Приложение Flask для веб-интерфейса (запущено на порту 8080).
- `templates/index.html`: HTML структура веб-интерфейса.
- `static/style.css`: Стили премиальной темной темы.
- `static/script.js`: Логика фронтенда.

## Результаты верификации
### Автоматический тест (CLI)
Запущен `main.py`, который загружает конфиг, регистрирует инструменты и отправляет тестовый промпт.
Подтверждена загрузка специфических инструментов SSH: `ssh_ensure_key`, `ssh_deploy_key`, `ssh_exec`.

### Верификация веб-интерфейса
Запущен сервер Flask на порту 8080.
**Команда:** `python3 app.py`

**Доступ:**
- URL: `http://localhost:8080`
- Интерфейс: Dark mode UI с полем ввода.
- **Функциональность:** Проверено через curl, что главная страница загружается и эндпоинт `/api/generate` возвращает валидный JSON от Gemini.
